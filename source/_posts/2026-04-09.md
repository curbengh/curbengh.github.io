---
title: Using Cloudflare Origin certificate on Salesforce
date: 2026-04-09
---

As certificate validity is now reducing to [200 days](https://www.ibm.com/think/insights/new-era-for-certificate-management) in 2026, Salesforce administrators must find a practical way to manage certificate renewal. Ideally, Salesforce should support [ACME protocol](https://en.wikipedia.org/wiki/Automatic_Certificate_Management_Environment) to automate renewal.

For those utilising Cloudflare CDN, it is possible to install Cloudflare [Origin certificate](https://developers.cloudflare.com/ssl/origin-configuration/origin-ca/) with 15-year validity on Salesforce. Although Salesforce [documentation](https://help.salesforce.com/s/articleView?id=platform.domain_mgmt_cert_prereqs.htm&type=5) mentions the certificate must be CA-signed, self-signed certificate including Cloudflare Origin certificate (which is not _public_ CA-signed) is actually acceptable.

1. In Cloudflare, change the [encryption mode](https://developers.cloudflare.com/ssl/origin-configuration/ssl-modes/full-strict/) to "Full (strict)".
2. Enable [True-Client-IP](https://developers.cloudflare.com/network/true-client-ip-header/) if on Enterprise plan.
3. Add CNAME record (if not exist) that points to the Salesforce site, e.g. `custom-domain.com.xxx.live.siteforce.com`. Ensure the CNAME record is in [Proxied/orange-clouded](https://developers.cloudflare.com/dns/proxy-status/) mode.
4. In Salesforce, generate and download certificate signing request (CSR).
5. In Cloudflare, create a new origin certificate, choose "Use my private key and CSR", paste the CSR value, update hostname and Create it.
6. Copy the PEM value then append [Origin CA](https://developers.cloudflare.com/ssl/static/origin_ca_rsa_root.pem) root certificate and save it with ".pem" extension. The file should have both leaf and root certificates.
7. Upload the ".pem" file to Salesforce.
8. Select the new certificate in Salesforce [domain configuration](https://help.salesforce.com/s/articleView?id=platform.domain_mgmt_cert_setup.htm&type=5).

In the last step, **do not** select the "Use a third-party service or CDN" [option](https://help.salesforce.com/s/articleView?id=platform.domain_mgmt_external_host_setup.htm&type=5) despite Cloudflare being a CDN service. That option will configure the Salesforce site to serve over HTTP, requiring "Flexible" encryption mode in Cloudflare which is not ideal.
